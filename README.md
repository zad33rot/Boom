# 💥 BOOM Messenger

BOOM — это современный и защищенный мессенджер реального времени, состоящий из десктопного клиента на базе Electron/React и быстрого бэкенда на FastAPI.

## 🌍 Живое демо и развертывание
**Проект уже развернут и работает на готовом сервере!**
В рамках проделанной работы по деплою:
- Был получен и настроен бесплатный домен от **DuckDNS**.
- Бэкенд и база данных успешно функционируют в сети.
- На сайт/сервер можно зайти прямо сейчас *https://boom-chat.duckdns.org/*.

## ✨ Ключевые возможности
- **Безопасная авторизация**: Вход и регистрация через подтверждение по Email (одноразовые OTP-коды). Пароли надежно хешируются с использованием `bcrypt`.
- **Шифрование данных**: Все сообщения шифруются перед сохранением в базу данных с использованием алгоритма симметричного шифрования (Fernet из библиотеки `cryptography`).
- **Реальное время**: Мгновенная доставка сообщений, реализованная через WebSockets.
- **Живые статусы**: Поддержка статусов «Онлайн» / «Офлайн», индикация того, что собеседник «печатает», а также отслеживание статуса прочтения сообщений (галочки).
- **Десктопное приложение**: Готовый клиент под десктоп с возможностью сборки установщика `.exe` (Electron Builder).
- **Кастомизация профиля**: Настройка юзернейма, никнейма, случайная генерация цветных аватаров.

## 🛠 Технологический стек

**Бэкенд:**
- Python 3
- FastAPI & Uvicorn (REST API + WebSockets)
- SQLite (локальная база данных `messenger.db`)
- Cryptography (Fernet)
- Smtplib (Отправка email через smtp.mail.ru)

**Фронтенд:**
- React 19 + Vite
- Electron
- CSS (чистый UI)

## 🚀 Установка и запуск (для локальной разработки)

### 1. Настройка Бэкенда

1. Перейдите в директорию `backend`:
   ```bash
   cd backend
   ```
2. Создайте и активируйте виртуальное окружение (рекомендуется):
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для Linux/Mac
   venv\Scripts\activate     # Для Windows
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Создайте файл `.env` в папке `backend` со следующими переменными окружения:
   ```env
   SMTP_EMAIL=ваш_email@mail.ru
   SMTP_PASSWORD=пароль_приложения_от_почты
   SECRET_KEY=ваш_сгенерированный_fernet_ключ
   ```
   *(Подсказка: Сгенерировать ключ можно выполнив команду в Python: `from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())`)*
5. Запустите сервер:
   ```bash
   uvicorn server:app --reload --host 0.0.0.0 --port 8000
   ```

### 2. Настройка Фронтенда (Десктоп-клиент)

1. Откройте новый терминал и перейдите в директорию `frontend`:
   ```bash
   cd frontend
   ```
2. Установите NPM-зависимости:
   ```bash
   npm install
   ```
3. Для запуска в режиме разработки (запускается Vite + Electron окно):
   ```bash
   npm run dev      # сначала можно запустить Vite для HMR
   npm run desktop  # затем запуск Electron (см. package.json)
   ```

### 3. Сборка приложения (.exe)

Чтобы собрать готовый исполняемый `.exe` файл для Windows, используйте команду:
```bash
npm run make-exe
```
После завершения процесса готовый инсталлятор будет находиться в директории `frontend/dist_electron/`.

## 📁 Структура проекта

В проекте реализовано четкое разделение на клиентскую (Frontend) и серверную (Backend) части. Ниже приведено детальное описание каждого файла.

### 🖥 Backend (Серверная часть - FastAPI)
- **`backend/server.py`** — Главный файл приложения. Содержит логику маршрутизации (API endpoints), авторизацию, работу с WebSockets для реального времени, функции отправки email с OTP-кодами и обработку SQL-запросов.
- **`backend/encryption.py`** — Модуль для шифрования и дешифрования текста сообщений с использованием библиотеки `cryptography` (алгоритм Fernet) и секретного ключа из `.env`.
- **`backend/auth.py`** — Дополнительный модуль для вынесения логики аутентификации (генерация кодов, проверка, хеширование паролей).
- **`backend/database.py`** — Инициализация и настройка подключения к базе данных.
- **`backend/models.py`** — Описание Pydantic-моделей (например, `EmailReq`, `CodeReq`, `LoginReq`), которые используются для валидации входящих JSON-запросов.
- **`backend/messenger.db`** — Локальный файл базы данных SQLite. В нем хранятся таблицы пользователей (с хешами паролей) и сообщений (в зашифрованном виде).
- **`backend/requirements.txt`** — Список всех Python-зависимостей, необходимых для работы сервера (например, `fastapi`, `bcrypt`, `cryptography`, `uvicorn`, `websockets`).
- **`backend/__pycache__/`** — Директория с предкомпилированными файлами `.pyc` для ускорения работы Python (создается автоматически).

### 🎨 Frontend (Клиентская часть - React + Electron)
- **`frontend/package.json`** — Главный конфигурационный файл NPM. Содержит зависимости (React, Electron, Vite) и скрипты для запуска (например, `npm run dev` и сборка `.exe` через `npm run make-exe`).
- **`frontend/package-lock.json`** — Файл фиксации точных версий всех установленных NPM-зависимостей для стабильной сборки.
- **`frontend/vite.config.js`** — Конфигурация Vite, сверхбыстрого сборщика, используемого вместо Webpack для React-приложения.
- **`frontend/electron-main.cjs`** — Точка входа для десктопного приложения Electron. Управляет созданием системного окна, загрузкой React-интерфейса и жизненным циклом десктоп-приложения.
- **`frontend/eslint.config.js`** — Правила линтера ESLint для проверки качества и стиля кода (ошибки React hooks и прочее).
- **`frontend/index.html`** — Главный HTML-шаблон, к которому подключается React (через элемент `<div id="root">`).
- **`frontend/boom.ico` / `boom.svg`** — Иконки приложения. `.ico` используется сборщиком (Electron Builder) для создания `.exe` файла и отображения в панели задач Windows.
- **`frontend/public/manifest.json`** — Манифест приложения с метаданными.

#### ⚛️ Исходный код React (`frontend/src/`)
- **`frontend/src/main.jsx`** — Главная точка входа React. Инициализирует рендер корневого компонента `App` в DOM-дерево.
- **`frontend/src/App.jsx`** — Корневой компонент. Выполняет роль маршрутизатора состояний: если пользователь не авторизован — показывает `<Auth />`, если авторизован — рендерит `<Chat />`.
- **`frontend/src/index.css`** — Глобальные каскадные стили приложения, определяющие дизайн и UI-элементы.
- **`frontend/src/components/Auth.jsx`** — Компонент страницы авторизации. Взаимодействует с API для отправки почты, ввода OTP, генерации паролей, проверки логина.
- **`frontend/src/components/Chat.jsx`** — Главный компонент мессенджера. Управляет списком контактов, окном сообщений, подключением к WebSockets, отправкой и приемом данных в реальном времени, поиском пользователей.
- **`frontend/src/assets/hero.png`** — Изображение/баннер, используемое для украшения интерфейса (например, на экране входа).
- **`frontend/src/assets/react.svg` / `vite.svg`** — Вспомогательные векторные иконки-логотипы библиотек.

### ⚙️ Общие конфигурационные файлы
- **`.gitignore`** — (Находится как в корне, так и в подпапках). Указывает системе контроля версий Git, какие папки и файлы следует игнорировать (например, `node_modules`, `venv/`, `.env`), чтобы они не попали в репозиторий.
